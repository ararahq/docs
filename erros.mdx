## Formato de erro

Todas as respostas de erro seguem o mesmo formato:

```json
{
  "error": {
    "code": "INSUFFICIENT_CREDITS",
    "message": "Your wallet does not have enough credits for this operation."
  }
}
```

| Campo | Tipo | Descricao |
|-------|------|-----------|
| `error.code` | string | Codigo maquina do erro (constante, pode ser usado em logica de retry). |
| `error.message` | string | Descricao legivel para humanos. |

## Codigos de erro comuns

### Autenticacao e autorizacao

| Codigo | HTTP | Descricao |
|--------|------|-----------|
| `INVALID_API_KEY` | 401 | API key invalida, expirada ou revogada. |
| `MISSING_API_KEY` | 401 | Header `Authorization` ausente. |
| `FORBIDDEN` | 403 | Chave valida, mas sem permissao para este recurso ou ambiente. |

### Validacao de input

| Codigo | HTTP | Descricao |
|--------|------|-----------|
| `INVALID_PHONE_NUMBER` | 400 | Numero de telefone em formato invalido. |
| `INVALID_EMAIL` | 400 | Endereco de email invalido. |
| `MISSING_REQUIRED_FIELD` | 400 | Campo obrigatorio ausente no body. |
| `INVALID_TEMPLATE_VARIABLES` | 400 | Variaveis do template nao conferem com o esperado. |
| `TEMPLATE_NOT_FOUND` | 404 | Template nao existe ou nao pertence a sua organizacao. |
| `MESSAGE_NOT_FOUND` | 404 | Mensagem nao encontrada. |
| `CAMPAIGN_NOT_FOUND` | 404 | Campanha nao encontrada. |
| `CONTACT_NOT_FOUND` | 404 | Contato nao encontrado. |

### Regras de negocio

| Codigo | HTTP | Descricao |
|--------|------|-----------|
| `INSUFFICIENT_CREDITS` | 402 | Saldo de creditos insuficiente. Recarregue sua carteira. |
| `TEMPLATE_NOT_APPROVED` | 422 | Template ainda nao foi aprovado pela Meta. |
| `TEMPLATE_PAUSED` | 422 | Template pausado por baixa qualidade. |
| `SESSION_WINDOW_CLOSED` | 422 | Janela de 24h fechada. Envie um template para reabrir a conversa. |
| `DUPLICATE_RESOURCE` | 409 | Recurso ja existe (ex: template com mesmo nome). |
| `IDEMPOTENCY_CONFLICT` | 409 | Requisicao com mesma `Idempotency-Key` ja processada com body diferente. |

### Rate limiting

| Codigo | HTTP | Descricao |
|--------|------|-----------|
| `RATE_LIMIT_EXCEEDED` | 429 | Limite de requisicoes excedido. Aguarde e tente novamente. |

### Erros internos

| Codigo | HTTP | Descricao |
|--------|------|-----------|
| `INTERNAL_ERROR` | 500 | Erro interno do servidor. Tente novamente ou contate suporte. |
| `PROVIDER_ERROR` | 502 | Erro no provider externo (Meta, email). Retry automatico em andamento. |
| `SERVICE_UNAVAILABLE` | 503 | Servico temporariamente indisponivel. |

## Escudo de Erros (WhatsApp)

A Arara traduz erros tecnicos da Meta em diagnosticos legiveis. Em vez de receber um codigo generico como `#131030`, voce recebe uma explicacao clara e uma sugestao de acao.

| Codigo Arara | Diagnostico | Acao recomendada |
|:-------------|:------------|:-----------------|
| `ararahq-63016` | Janela de conversacao fechada. | Envie um template para reabrir a conversa. |
| `ararahq-insufficient-funds` | Saldo de creditos esgotado. | Recarregue sua carteira no painel. |
| `ararahq-rate-limit` | Limite de requisicoes excedido. | Reduza a velocidade ou use a API de Campanhas. |
| `ararahq-template-mismatch` | Variaveis do template invalidas. | Verifique os parametros no JSON. |
| `ararahq-media-invalid` | Arquivo pesado ou nao suportado. | Reduza o tamanho do arquivo (max 16MB/100MB). |
| `ararahq-template-error` | Template rejeitado ou pausado. | Verifique o status no Gerenciador da Meta. |
| `ararahq-invalid-number` | Numero invalido ou sem WhatsApp. | Verifique o formato (DDI + DDD + numero). |
| `ararahq-meta-block` | Bloqueio preventivo de spam. | Aguarde ou mude o conteudo da mensagem. |
| `ararahq-opt-out` | O destinatario bloqueou a empresa. | Remova o contato da sua lista de envios. |
| `ararahq-generic` | Erro generico no processamento. | Tente reenviar em alguns minutos. |

## Como consultar o erro de uma mensagem

Ao consultar uma mensagem com status `FAILED`, o campo `failureReason` contem o codigo do erro:

```bash
curl https://new-api.ararahq.com/api/v2/messages/ara_msg_k9Bf2XqP \
  -H "Authorization: Bearer ara_live_xxx"
```

```json
{
  "id": "ara_msg_k9Bf2XqP",
  "status": "FAILED",
  "failureReason": "ararahq-63016",
  "failureMessage": "Janela de conversacao fechada. Envie um template para reabrir a conversa.",
  "createdAt": "2026-04-04T14:30:00Z"
}
```

## Retentativas automaticas

Para erros temporarios (instabilidade da Meta, timeouts), a Arara aplica retry com backoff exponencial automaticamente antes de marcar a mensagem como `FAILED`. Voce nao precisa implementar retry no seu lado para esses casos.

<Accordion title="V1 Legacy">
  Na V1, o formato de erro era diferente:

  ```json
  {
    "error": "Unauthorized",
    "message": "Invalid or expired API key"
  }
  ```

  A V2 padroniza todos os erros no formato `{ "error": { "code": "...", "message": "..." } }`.
</Accordion>
