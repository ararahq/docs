A Arara expõe um servidor MCP (Model Context Protocol) que dá a qualquer agente de IA acesso completo à sua conta: mensagens, templates, campanhas, conversas, base de conhecimento e recuperação de receita via AbacatePay.

## Como Conectar

<Tabs>
  <Tab title="Claude Desktop">
    Edite o arquivo `claude_desktop_config.json`:

    ```json
    {
      "mcpServers": {
        "ararahq": {
          "url": "https://mcp.ararahq.com/sse",
          "headers": {
            "X-Arara-Key": "SUA_API_KEY_ARARA",
            "X-Abacate-Key": "SUA_API_KEY_ABACATEPAY"
          }
        }
      }
    }
    ```

    <Note>
      `X-Abacate-Key` é opcional. Necessário apenas para as ferramentas de revenue recovery (AbacatePay).
    </Note>
  </Tab>

  <Tab title="Cursor">
    Acesse **Settings → Models → MCP Servers** e adicione:

    - **Name**: AraraHQ
    - **Type**: `SSE`
    - **URL**: `https://mcp.ararahq.com/sse`
    - **Header** `X-Arara-Key`: sua API key da Arara
  </Tab>

  <Tab title="Windsurf">
    Edite o arquivo `.windsurf/mcp.json` na raiz do projeto:

    ```json
    {
      "servers": {
        "ararahq": {
          "url": "https://mcp.ararahq.com/sse",
          "headers": {
            "X-Arara-Key": "SUA_API_KEY_ARARA"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="Modo Local (stdio)">
    Para rodar o servidor localmente via npx:

    ```json
    {
      "mcpServers": {
        "ararahq": {
          "command": "npx",
          "args": ["-y", "@ararahq/mcp"],
          "env": {
            "ARARA_API_KEY": "SUA_API_KEY_ARARA",
            "ABACATEPAY_API_KEY": "SUA_API_KEY_ABACATEPAY",
            "ARARA_BASE_URL": "https://new-api.ararahq.com/api"
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

<Note>
  **Usando a V2 da AraraHQ?** No modo local (stdio), adicione `"ARARA_BASE_URL": "https://new-api.ararahq.com/api"` no bloco `env`. Nos modos SSE (Claude Desktop, Cursor, Windsurf), o servidor remoto `mcp.ararahq.com` já roteia automaticamente com base na sua API key.
</Note>

## Ferramentas Disponíveis

### Conta e Configuração

| Ferramenta | O que faz |
| :--- | :--- |
| `get_account_overview` | Saldo da carteira, métricas de entrega (sent / delivered / read / failed) e spend total |
| `manage_organization` | Gerencia números ativos, configuração de webhook, brain config e membros do time |
| `manage_api_keys` | Lista, cria e revoga API keys da conta |
| `configure_guardian_policy` | Define regras customizadas de bloqueio de conteúdo para a sessão (palavras proibidas, padrões de marca) |

### Mensagens

| Ferramenta | O que faz |
| :--- | :--- |
| `send_smart_message` | Envia mensagem única para um número (texto, template ou mídia). Guardian Mode ativo. |
| `send_batch_messages` | Envia até 1.000 mensagens em uma única chamada. Guardian Mode ativo em cada mensagem. |
| `manage_messages` | Lista mensagens do dashboard com filtros ou consulta o status de um ID específico |

### Templates

| Ferramenta | O que faz |
| :--- | :--- |
| `manage_templates` | Cria, lista, consulta status de aprovação, analytics por template e deleta templates |

### Campanhas

| Ferramenta | O que faz |
| :--- | :--- |
| `create_campaign` | Cria e dispara campanha com template + lista de destinatários e variáveis por contato |
| `monitor_campaign_responses` | Busca conversas abertas desde o disparo, classifica cada resposta por urgência (URGENT / COMPLAINT / QUESTION / POSITIVE / ROUTINE) e retorna fila de triage |

### Conversas

| Ferramenta | O que faz |
| :--- | :--- |
| `manage_conversations` | Lista conversas abertas, acessa histórico completo de uma conversa e envia reply. Guardian Mode ativo no reply. |

### Conteúdo e Memória

| Ferramenta | O que faz |
| :--- | :--- |
| `upload_media` | Sobe arquivo de uma URL para o CDN da Arara e retorna short URL |
| `manage_knowledge_base` | Lista, salva, atualiza e deleta entradas da base de conhecimento do brain |
| `build_business_memory` | Analisa histórico de conversas (sentimento, humor, urgência), calcula LTV via AbacatePay, classifica o tier do cliente (NEW / ENGAGED / VIP) e salva na base de conhecimento |

### Revenue Recovery (AbacatePay)

| Ferramenta | O que faz |
| :--- | :--- |
| `autonomous_recovery` | Escaneia todos os leaks do AbacatePay, enriquece cada um com histórico de mensagens do cliente, calcula total de R$ em risco e retorna briefing pronto para ação imediata |
| `atomic_negotiation_cycle` | Cria produto + checkout com desconto + envia o link via WhatsApp em uma única chamada atômica |
| `confirm_payment_handshake` | Verifica o status real de um checkout em tempo real |
| `check_revenue_leaks` | Scan rápido de checkouts pendentes ou vencidos no AbacatePay |
| `negotiate_payment` | Cria produto e checkout no AbacatePay e retorna o link (sem enviar) |

## Os 5 Pilares

<Steps>
  <Step title="Autonomous Revenue Recovery">
    `autonomous_recovery` monitora leaks (Pix vencido, falha de cartão), busca o histórico de cada cliente e entrega ao agente um briefing completo: quanto está em risco, qual o humor do cliente, qual a janela de oportunidade. O agente age sem esperar ordem.
  </Step>
  <Step title="Atomic Negotiation">
    `atomic_negotiation_cycle` executa o ciclo completo em uma chamada: cria o produto com desconto, gera o checkout no AbacatePay e envia o link via WhatsApp. O diálogo vira o contrato.
  </Step>
  <Step title="Guardian Mode (Brand Safety)">
    Todas as ferramentas de envio passam pelo Guardian antes de qualquer mensagem sair. `configure_guardian_policy` permite definir regras customizadas por sessão: palavras proibidas, padrões de marca, dados sensíveis. Nada escapa.
  </Step>
  <Step title="Business Memory Layer">
    `build_business_memory` constrói um perfil profundo: sentimento das últimas conversas, LTV histórico, tier do cliente, janela de 24h aberta ou não. A IA sabe com quem está falando antes de falar.
  </Step>
  <Step title="Mass Orchestration">
    `create_campaign` + `monitor_campaign_responses` formam o ciclo completo: dispara segmentado, lê o aftermath, classifica respostas por urgência, sugere auto-reply para POSITIVE e QUESTION, escala URGENT e COMPLAINT para humano.
  </Step>
</Steps>

## Exemplos de Uso

<AccordionGroup>
  <Accordion title="Recuperação de receita autônoma" icon="money-bill-trend-up">
    ```
    "Verifique se temos algum leak de receita do AbacatePay. Para os 3 maiores valores,
    analise o perfil de cada cliente e me dê um plano de ação para recuperar hoje."
    ```

    O agente chama `autonomous_recovery`, recebe o briefing por cliente com valor em risco,
    histórico de conversas e sugestão de abordagem. Sem precisar de mais contexto.
  </Accordion>

  <Accordion title="Negociação atômica no WhatsApp" icon="handshake">
    ```
    "O cliente (11) 99999-9999 achou o valor alto. Gere um link com 15% de desconto
    e mande no WhatsApp dele explicando a oferta."
    ```

    Uma chamada para `atomic_negotiation_cycle` cria o produto, gera o checkout e envia.
    Use `confirm_payment_handshake` para checar confirmação.
  </Accordion>

  <Accordion title="Campanha com gestão de respostas" icon="bullhorn">
    ```
    "Dispare a campanha de reativação para a lista de clientes inativos.
    Após 2 horas, me traga uma triagem das respostas priorizando os mais urgentes."
    ```

    `create_campaign` dispara. Depois, `monitor_campaign_responses` retorna a fila
    classificada por URGENT → COMPLAINT → QUESTION → POSITIVE → ROUTINE.
  </Accordion>

  <Accordion title="Perfil de cliente antes de responder" icon="brain">
    ```
    "Antes de eu responder o cliente (21) 88888-8888, me dê o perfil completo dele:
    humor das últimas interações, histórico de pagamentos e se a janela de 24h está aberta."
    ```

    `build_business_memory` cruza histórico de conversas com AbacatePay e retorna
    tier, mood score, LTV e janela de oportunidade.
  </Accordion>

  <Accordion title="Gestão operacional da conta" icon="gear">
    ```
    "Me dê um resumo da conta: saldo, taxa de entrega da última semana
    e quais templates estão aprovados e prontos para uso."
    ```

    `get_account_overview` + `manage_templates` com `action: list`.
  </Accordion>
</AccordionGroup>

## SDK vs MCP

| | **Node SDK** | **MCP Server** |
| :--- | :--- | :--- |
| **Quando usar** | Backends, scripts, automações fixas | Agentes de IA, Claude, Cursor, Windsurf |
| **Como integra** | `npm install @ararahq/sdk` | Configuração JSON no cliente de IA |
| **Controle** | Total — você define cada chamada | Automático — a IA descobre e usa as tools |
| **Melhor para** | Pipelines programáticos | Orquestração por linguagem natural |

## Segurança

- **Chaves não armazenadas**: o servidor `mcp.ararahq.com` não persiste suas chaves. Elas são usadas apenas durante a sessão ativa via header.
- **Guardian Mode obrigatório**: todas as ferramentas de envio passam por inspeção antes de qualquer mensagem sair.
- **Escopo por chave**: o MCP opera com as mesmas permissões da API key fornecida. Use chaves com escopo mínimo necessário.

<Warning>
  Nunca compartilhe sua `X-Arara-Key` publicamente. Gere chaves dedicadas para cada agente ou ambiente no painel da Arara.
</Warning>
