A Arara expõe um servidor MCP (Model Context Protocol) que dá a qualquer agente de IA acesso completo à sua conta: mensagens, templates, campanhas, conversas, base de conhecimento e recuperação de receita via AbacatePay.

## Como Conectar

<Tabs>
  <Tab title="Claude Desktop">
    Edite o arquivo `claude_desktop_config.json`:

    ```json
    {
      "mcpServers": {
        "ararahq": {
          "url": "https://mcp.ararahq.com/sse",
          "headers": {
            "X-Arara-Key": "SUA_API_KEY_ARARA",
            "X-Abacate-Key": "SUA_API_KEY_ABACATEPAY"
          }
        }
      }
    }
    ```

    <Note>
      `X-Abacate-Key` é opcional. Necessário apenas para as ferramentas de revenue recovery (AbacatePay).
    </Note>
  </Tab>

  <Tab title="Cursor">
    Acesse **Settings → Models → MCP Servers** e adicione:

    - **Name**: AraraHQ
    - **Type**: `SSE`
    - **URL**: `https://mcp.ararahq.com/sse`
    - **Header** `X-Arara-Key`: sua API key da Arara
  </Tab>

  <Tab title="Windsurf">
    Edite o arquivo `.windsurf/mcp.json` na raiz do projeto:

    ```json
    {
      "servers": {
        "ararahq": {
          "url": "https://mcp.ararahq.com/sse",
          "headers": {
            "X-Arara-Key": "SUA_API_KEY_ARARA"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="Modo Local (stdio)">
    Para rodar o servidor localmente via npx:

    ```json
    {
      "mcpServers": {
        "ararahq": {
          "command": "npx",
          "args": ["-y", "ararahq-mcp"],
          "env": {
            "ARARA_API_KEY": "SUA_API_KEY_ARARA",
            "ABACATEPAY_API_KEY": "SUA_API_KEY_ABACATEPAY",
            "ARARA_BASE_URL": "https://api.ararahq.com/api"
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

`list_conversations`, `get_conversation_messages`, `reply_in_conversation`, `check_window_status`, `bulk_window_check`, `get_lead_stats`

`brain_interact`, `brain_suggest_reply`, `get_brain_config`, `get_brain_metrics`, `add_brain_knowledge`, `list_brain_knowledge`, `update_brain_knowledge`, `delete_brain_knowledge`, `ingest_url_to_brain`

## Os 5 pilares

<Steps>
  <Step title="Revenue Recovery">
    `find_revenue_leaks` escaneia os checkouts pendentes/vencidos do AbacatePay e retorna a fila com valor em risco. Combinado com `negotiate_payment` (cria checkout com desconto) e `check_payment_status`, o agente fecha o ciclo de cobrança sem você abrir o app.
  </Step>
  <Step title="Negociação no WhatsApp">
    `negotiate_payment` gera o checkout e devolve o link. Combina com `send_message` (template UTILITY) pra mandar pro cliente. `check_payment_status` confirma quando paga.
  </Step>
  <Step title="Guardian Mode (Brand Safety)">
    Todas as ferramentas de envio passam pelo Guardian antes de qualquer mensagem sair. `configure_guardian_policy` define regras por sessão (palavras proibidas, padrões de marca, PII). Violações vão pra `list_policy_violations`.
  </Step>
  <Step title="Brain Memory">
    `brain_interact` consulta o agente RAG com o histórico da org. `add_brain_knowledge` + `ingest_url_to_brain` carregam contexto novo. `brain_suggest_reply` propõe resposta pra uma conversa específica.
  </Step>
  <Step title="Orquestração em massa">
    `create_campaign` dispara segmentado. `list_conversations` + `get_conversation_messages` leem o aftermath. `reply_in_conversation` responde individualmente quando o agente decide.
  </Step>
</Steps>

## Exemplos de uso

<AccordionGroup>
  <Accordion title="Recuperação de receita autônoma" icon="money-bill-trend-up">
    ```
    "Roda find_revenue_leaks. Pros 3 maiores valores em risco hoje, traz histórico recente
    do cliente via get_conversation_messages e me dá um plano de abordagem."
    ```

    O agente chama `find_revenue_leaks`, cruza com `get_conversation_messages` por telefone,
    e sintetiza recomendação. Você revisa e aprova o envio.
  </Accordion>

  <Accordion title="Negociação no WhatsApp" icon="handshake">
    ```
    "Cliente +5511999... achou o valor alto. Gere checkout com 15% off via negotiate_payment
    e manda o link via send_message usando o template 'oferta_personalizada'."
    ```

    Um `negotiate_payment` cria o checkout, `send_message` entrega o link. Depois `check_payment_status`
    confirma quando o cliente paga.
  </Accordion>

  <Accordion title="Campanha com triagem de respostas" icon="bullhorn">
    ```
    "Dispare a campanha de reativação via create_campaign. Depois de 2h, traga as

    ```

    `create_campaign` dispara. `list_conversations` traz as que tiveram resposta.
    O agente lê cada uma via `get_conversation_messages` e classifica.
  </Accordion>

    Três chamadas, agente sintetiza tier, mood, janela e contexto.
  </Accordion>

  <Accordion title="Resumo operacional da conta" icon="gear">

## SDK vs MCP

| | **Node SDK** | **MCP Server** |
| :--- | :--- | :--- |
| **Quando usar** | Backends, scripts, automações fixas | Agentes de IA, Claude, Cursor, Windsurf |
| **Como integra** | `npm install @ararahq/sdk` | Configuração JSON no cliente de IA |
| **Controle** | Total — você define cada chamada | Automático — a IA descobre e usa as tools |
| **Melhor para** | Pipelines programáticos | Orquestração por linguagem natural |

## Segurança

- **Chaves não armazenadas**: o servidor `mcp.ararahq.com` não persiste suas chaves. Elas são usadas apenas durante a sessão ativa via header.
- **Guardian Mode obrigatório**: todas as ferramentas de envio passam por inspeção antes de qualquer mensagem sair.
- **Escopo por chave**: o MCP opera com as mesmas permissões da API key fornecida. Use chaves com escopo mínimo necessário.

<Warning>
  Nunca compartilhe sua `X-Arara-Key` publicamente. Gere chaves dedicadas para cada agente ou ambiente no painel da Arara.
</Warning>
