O Brain não responde só com texto. Ele **executa**.

Brain Actions são integrações nativas que o agente AI da Arara pode invocar durante uma conversa do WhatsApp — sem você precisar escrever código de orquestração. O cliente pede uma fatura, o agente cria o Pix no AbacatePay e envia o link. Pede um horário, o agente checa o Cal.com e confirma. Pergunta sobre produto, o agente busca no catálogo e responde com a ficha.

<Note>
  Brain Actions só rodam dentro de uma sessão ativa do Brain (canUseAi = true). Se o seu plano não inclui Brain, [adicione o add-on](/billing#brain-addon) primeiro.
</Note>

## Como funciona

Toda Action segue o mesmo ciclo:

<Steps>
  <Step title="Você habilita a Action">
    No dashboard em **Brain → Actions**, ou via `PATCH /v1/brain/actions/{actionType}` configurando os segredos (API key do provider, IDs internos).
  </Step>
  <Step title="O agente entende quando chamar">
    Cada Action declara um schema de quando deve ser invocada (ex: "quando o cliente confirma compra mas não tem método de pagamento"). O modelo decide na conversa.
  </Step>
  <Step title="A Arara executa">
    A Action roda no nosso backend, com os segredos criptografados. O resultado volta como contexto pro modelo.
  </Step>
  <Step title="O agente responde no WhatsApp">
    Texto + link/botão/mídia gerado pela Action. Tudo passa pelo [Guardian Mode](/concepts/guardian) antes de sair.
  </Step>
</Steps>

## Configurando via API

```bash
curl -X PATCH https://api.ararahq.com/api/v1/brain/actions/abacate-pix \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "enabled": true,
    "config": {
      "apiKey": "abacate_live_xxx",
      "defaultExpirationMinutes": 60
    }
  }'
```

Os segredos (`apiKey`, tokens OAuth) ficam criptografados — não voltam em nenhum `GET`. Pra rodar trocar você precisa reenviar o valor.

## Por que isso importa

Sem Actions, um agente AI é um chatbot — responde, mas não resolve. Com Actions, a conversa **vira o contrato**: cliente pediu, agente cobrou, pagou na hora. Esse é o desbloqueio real do Revenue OS — fechar receita dentro do canal onde o cliente já está.
