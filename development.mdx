## Boas-vindas aos Contribuidores!

Este guia detalha os passos para configurar seu ambiente de desenvolvimento e começar a contribuir com a documentação da Arara API.

<Card title="Comece em Minutos" icon="hourglass-start">
  Siga os passos abaixo para ter o ambiente de preview da documentação rodando localmente.
</Card>

## Pré-requisitos

<Info>
  **Pré-requisitos**:

  - Node.js (versão 19 ou superior)
  - Um clone local do repositório da documentação da Arara (via GitHub).
</Info>

## Passo a Passo: Configuração do Ambiente

<Steps>
  <Step title="Instalação Global da CLI">
    A CLI do Mintlify permite que você execute a documentação localmente e utilize ferramentas de validação.

    <CodeGroup>

    ```bash npm
    npm install -g mint
    ```

    
    ```bash yarn
    yarn global add mint
    ```

    </CodeGroup>
  </Step>
  <Step title="Inicie o Servidor de Preview">
    Navegue até o diretório raiz da sua documentação (onde o arquivo `docs.json` está localizado) e execute o seguinte comando:

    ```bash
    mint dev
    ```

    Um preview local da sua documentação estará disponível em `http://localhost:3000`.

    <Tip>
      O preview é atualizado automaticamente sempre que você salva um arquivo, sem precisar reiniciar o servidor.
    </Tip>
  </Step>
</Steps>

## Configurações Avançadas

### Portas Personalizadas

Por padrão, o Mintlify utiliza a porta `3000`. Você pode customizá-la usando a flag `--port`. Por exemplo, para rodar na porta `3333`:

```bash
mint dev --port 3333
```

Se a porta escolhida já estiver em uso, o Mintlify tentará a próxima porta disponível:

```md
Port 3000 is already in use. Trying 3001 instead.
```

### Versões do Mintlify CLI

Cada release da CLI está associada a uma versão específica do Mintlify. Se o seu preview local não estiver alinhado com a versão de produção, atualize a CLI:

<CodeGroup>

```bash npm
npm install -g mint@latest
```


```bash yarn
yarn global upgrade mint
```

</CodeGroup>

### Validação de Links

A CLI pode ajudar a identificar links quebrados em sua documentação. Para verificar, use o comando:

```bash
mint broken-links
```

## Ferramentas e Troubleshooting

### Formatação de Código

Recomendamos o uso de extensões em seu IDE para reconhecer e formatar arquivos MDX. Para usuários do VS Code, considere:

- [**MDX VSCode extension**](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para _syntax highlighting_.
- [**Prettier**](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para formatação de código consistente.

### Solução de Problemas (Troubleshooting)

<AccordionGroup>
  <Accordion title='Erro: Não foi possível carregar o módulo "sharp" com o runtime darwin-arm64'>
    Este erro pode ser devido a uma versão desatualizada do Node.js. Tente o seguinte:

    1. Remova a versão atualmente instalada da CLI: `npm remove -g mint`
    2. Atualize para o Node.js v19 ou superior.
    3. Reinstale a CLI: `npm install -g mint`
  </Accordion>
  <Accordion title="Problema: Erro desconhecido">
    **Solução:** Vá para a raiz do seu sistema e delete a pasta `~/.mintlify`. Em seguida, execute `mint dev` novamente.
  </Accordion>
</AccordionGroup>

Curioso sobre o que mudou na última versão da CLI? [Confira o changelog da CLI.](https://www.npmjs.com/package/mint?activeTab=versions)
