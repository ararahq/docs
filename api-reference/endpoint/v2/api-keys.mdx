Gerencie as chaves que suas aplicacoes usam para autenticar na API.

**Base URL:** `https://new-api.ararahq.com/api/v2`

<Warning>
  Toda chave e exibida **uma unica vez**, no momento da criacao. A Arara nao armazena o valor completo â€” se voce perder, cria uma nova e revoga a antiga.
</Warning>

---

## Criar uma API key

`POST /v2/api-keys`

```bash
curl -X POST https://new-api.ararahq.com/api/v2/api-keys \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{ "mode": "LIVE" }'
```

### Parametros

| Campo | Tipo | Obrigatorio | Descricao |
|-------|------|-------------|-----------|
| `mode` | string | nao | `LIVE` (producao) ou `TEST` (sandbox). Default `LIVE`. |

Resposta (`201 Created`):

```json
{
  "id": "7f8a1c2d-4e5f-6789-abcd-ef0123456789",
  "key": "ara_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "mode": "LIVE",
  "createdAt": "2026-04-13T18:30:00Z"
}
```

| Campo | Descricao |
|-------|-----------|
| `id` | UUID da chave. Use pra revogar depois. |
| `key` | **Valor completo.** Aparece apenas aqui. Copie e guarde com seguranca. |
| `mode` | `LIVE` ou `TEST`. |
| `createdAt` | Data de criacao. |

## Listar chaves ativas

`GET /v2/api-keys`

Retorna um array simples com todas as chaves ativas (nao revogadas) da organizacao.

```bash
curl "https://new-api.ararahq.com/api/v2/api-keys?mode=LIVE" \
  -H "Authorization: Bearer ara_live_xxx"
```

### Query params

| Param | Tipo | Descricao |
|-------|------|-----------|
| `mode` | string | Opcional. Filtra por `LIVE` ou `TEST`. Omitir retorna ambos. |

Resposta (`200 OK`):

```json
[
  {
    "id": "7f8a1c2d-4e5f-6789-abcd-ef0123456789",
    "lastFour": "a3f9",
    "mode": "LIVE",
    "lastUsedAt": "2026-04-13T18:25:00Z",
    "createdAt": "2026-03-01T10:00:00Z"
  },
  {
    "id": "8a2b3c4d-5e6f-7890-abcd-ef1234567890",
    "lastFour": "b7e2",
    "mode": "TEST",
    "lastUsedAt": null,
    "createdAt": "2026-03-15T12:00:00Z"
  }
]
```

### Campos

| Campo | Tipo | Descricao |
|-------|------|-----------|
| `id` | UUID | Identificador da chave. |
| `lastFour` | string | Ultimos 4 caracteres da chave (para identificar sem expor o valor). |
| `mode` | string | `LIVE` ou `TEST`. |
| `lastUsedAt` | string \| null | Quando foi usada pela ultima vez. `null` se nunca foi usada. |
| `createdAt` | string | Data de criacao. |

## Revogar uma chave

`DELETE /v2/api-keys/{id}`

```bash
curl -X DELETE https://new-api.ararahq.com/api/v2/api-keys/7f8a1c2d-4e5f-6789-abcd-ef0123456789 \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta (`204 No Content`) â€” sem body.

<Warning>
  Revogacao e **imediata e irreversivel**. Qualquer requisicao usando essa chave passa a retornar `401 Unauthorized` instantaneamente.
</Warning>

---

## Erros

| HTTP | Quando |
|------|--------|
| `400` | `mode` fora de `LIVE`/`TEST`. |
| `401` | Chave de autenticacao ausente ou invalida. |
| `403` | Tentativa de revogar chave de outra organizacao. |
| `404` | Chave nao encontrada (no DELETE). |
