Flows sao automações criadas no Dashboard da Arara. Voce pode dispara-los programaticamente via API quando um evento acontece no seu sistema (ex: usuario se cadastrou, pagamento confirmado).

## Disparar um flow

`POST /v2/flows/{internalId}/trigger`

<CodeGroup>

```bash cURL
curl -X POST https://new-api.ararahq.com/api/v2/flows/ara_flow_abc123/trigger \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "5511999999999",
    "variables": {
      "name": "Joao",
      "orderId": "12345"
    }
  }'
```

```javascript Node.js
const response = await fetch(
  "https://new-api.ararahq.com/api/v2/flows/ara_flow_abc123/trigger",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer ara_live_xxx",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      to: "5511999999999",
      variables: { name: "Joao", orderId: "12345" },
    }),
  }
);

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    "https://new-api.ararahq.com/api/v2/flows/ara_flow_abc123/trigger",
    headers={
        "Authorization": "Bearer ara_live_xxx",
        "Content-Type": "application/json",
    },
    json={
        "to": "5511999999999",
        "variables": {"name": "Joao", "orderId": "12345"},
    },
)

print(response.json())
```

</CodeGroup>

### Parametros

| Campo | Tipo | Obrigatorio | Descricao |
|-------|------|-------------|-----------|
| `to` | string | Sim | Destinatario (telefone com DDI ou email, depende do flow). |
| `variables` | object | Nao | Variaveis usadas nos templates do flow. |

### Response `202 Accepted`

```json
{
  "id": "ara_exec_xyz789",
  "flowId": "ara_flow_abc123",
  "to": "5511999999999",
  "status": "TRIGGERED",
  "createdAt": "2026-04-04T14:30:00Z"
}
```

## Testar um flow (dry-run)

`POST /v2/flows/{internalId}/test`

Executa o flow em modo de teste. Nenhuma mensagem e enviada e nenhum credito e consumido. Util para validar a configuracao antes de ir para producao.

```bash
curl -X POST https://new-api.ararahq.com/api/v2/flows/ara_flow_abc123/test \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "5511999999999",
    "variables": {
      "name": "Joao",
      "orderId": "12345"
    }
  }'
```

### Parametros

Mesmos parametros do endpoint `/trigger`.

### Response `200 OK`

```json
{
  "id": "ara_exec_test_456",
  "flowId": "ara_flow_abc123",
  "to": "5511999999999",
  "status": "TEST_COMPLETED",
  "steps": [
    {
      "step": 1,
      "type": "SEND_TEMPLATE",
      "template": "welcome_flow",
      "status": "SIMULATED"
    },
    {
      "step": 2,
      "type": "WAIT",
      "duration": "24h",
      "status": "SIMULATED"
    },
    {
      "step": 3,
      "type": "SEND_TEMPLATE",
      "template": "followup_flow",
      "status": "SIMULATED"
    }
  ],
  "estimatedCost": 1.20,
  "createdAt": "2026-04-04T14:30:00Z"
}
```

A resposta inclui todos os passos do flow e o custo estimado, permitindo validar a logica sem efeitos colaterais.
