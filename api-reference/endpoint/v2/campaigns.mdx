Campanhas são o jeito certo de disparar em massa. Diferente de loops em `POST /v1/messages`, a campanha **valida saldo antecipado, enfileira tudo, processa em background com rate limit respeitado, e expõe métricas agregadas**.

**Base URL:** `https://api.ararahq.com`

<Note>
  Pra **1 mensagem**: [`POST /v1/messages`](/api-reference/endpoint/v2/messages).<br/>
  Pra **batch de até 1000** com payload simples: [`POST /v1/messages/batch`](/api-reference/endpoint/v2/messages#lote).<br/>
  Pra **disparo segmentado em massa**: `POST /v1/campaigns`.
</Note>

## Criar campanha

`POST /v1/campaigns`

```bash
curl -X POST https://api.ararahq.com/v1/campaigns \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Idempotency-Key: bf26-main" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Black Friday 2026",
    "templateName": "bf_oferta_geral",
    "contacts": [
      { "to": "+5511999999999", "variables": ["João", "30"] },
      { "to": "+5511888888888", "variables": ["Maria", "25"] }
    ]
  }'
```

Resposta (`200 OK`):

```json
{
  "id": "7b3789fb-4c5d-6789-abcd-ef0123456789",
  "name": "Black Friday 2026",
  "status": "PROCESSING",
  "totalMessages": 2,
  "totalCost": 0.80
}
```

<Note>
  Resposta é minimal por design — pra detalhes (sent/delivered/timing), consulte `GET /v1/campaigns/{id}` que tem o `CampaignDetailResponse` completo.
</Note>

### Parâmetros

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `name` | string | sim | Identificador interno (não vai pra Meta) |
| `templateName` | string | sim | Template aprovado pela Meta |
| `sender` | string | não | Número específico pra usar (default: round-robin da org) |
| `contacts` | array | sim | Destinatários. Min 1 |
| `contacts[].to` | string | sim | E.164 (`+5511999999999`) |
| `contacts[].variables` | string[] | não | Array posicional pra `{{1}}`, `{{2}}`... do template. Default: array vazio |

<Warning>
  `contacts[].variables` é **array posicional** (`["João", "30"]`), não objeto. Cliente que enviar `{"name": "João"}` recebe erro de bind.
</Warning>

### Idempotency

`Idempotency-Key` é **obrigatório**. Request sem o header retorna 400. Gere UUID único por campanha — se o mesmo header chegar de novo, a Arara devolve a campanha original (sem duplicar custo).

```bash
-H "Idempotency-Key: $(uuidgen)"
```

## Consultar campanha

`GET /v1/campaigns/{id}`

```bash
curl https://api.ararahq.com/v1/campaigns/7b3789fb-4c5d-6789-abcd-ef0123456789 \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta (`200 OK`):

```json
{
  "id": "7b3789fb-4c5d-6789-abcd-ef0123456789",
  "name": "Black Friday 2026",
  "status": "COMPLETED",
  "templateName": "bf_oferta_geral",
  "templateBody": "Oi {{1}}, oferta {{2}}% off válida hoje!",
  "totalMessages": 50000,
  "sentCount": 48500,
  "deliveredCount": 47200,
  "readCount": 38900,
  "clickedCount": 4120,
  "convertedCount": 312,
  "convertedValue": 87650.00,
  "totalCost": 18647.50,
  "scheduledAt": null,
  "startedAt": "2026-11-29T18:00:12Z",
  "finishedAt": "2026-11-29T18:23:14Z",
  "createdAt": "2026-11-29T18:00:00Z"
}
```

`clickedCount` e `convertedCount` dependem de integração com [Smart Links](/smart-links).

## Listar campanhas

`GET /v1/campaigns`

```bash
curl "https://api.ararahq.com/v1/campaigns?page=0&size=20" \
  -H "Authorization: Bearer ara_live_xxx"
```

Query params:

| Param | Tipo | Default | Descrição |
|---|---|---|---|
| `page` | integer | `0` | Página |
| `size` | integer | `20` | Itens por página (1-100) |
| `status` | string | — | Filtra por status (`PROCESSING`, `COMPLETED`, `CANCELLED`, `FAILED`) |

Resposta usa o envelope `content / totalElements / totalPages` do Spring Page:

```json
{
  "content": [
    {
      "id": "7b3789fb-4c5d-6789-abcd-ef0123456789",
      "name": "Black Friday 2026",
      "status": "COMPLETED",
      "templateName": "bf_oferta_geral",
      "totalMessages": 50000,
      "sentCount": 48500,
      "totalCost": 18647.50,
      "createdAt": "2026-11-29T18:00:00Z"
    }
  ],
  "totalPages": 1,
  "totalElements": 1
}
```

## Estimar custo antes

`GET /v1/campaigns/estimate`

Calcula custo sem criar a campanha — bom pra UX "Confirma envio? Total R$ X".

```bash
curl "https://api.ararahq.com/v1/campaigns/estimate?templateName=bf_oferta_geral&count=50000" \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta:

```json
{
  "templateCategory": "MARKETING",
  "recipientCount": 50000,
  "templateCost": 0.295,
  "araraFee": 0.0885,
  "unitPrice": 0.3835,
  "totalCost": 19175.00
}
```

| Campo | Descrição |
|---|---|
| `templateCost` | Custo Meta por mensagem entregue |
| `araraFee` | Margem Arara por mensagem |
| `unitPrice` | `templateCost + araraFee` |
| `totalCost` | `unitPrice × recipientCount` |

Pra verificar saldo antes do disparo, consulte separadamente [Wallet](/concepts/pricing).

## Status

| Status | Significado |
|---|---|
| `PROCESSING` | Disparando contatos |
| `COMPLETED` | Todos contatos processados (entregues ou falharam) |
| `CANCELLED` | Cancelado manualmente |
| `FAILED` | Falha geral (saldo insuficiente antes de iniciar, etc) |

Pra acompanhar **mensagem por mensagem em tempo real**, escute o webhook `message.status_updated`. Campanha terminada dispara `campaign.completed`.

## Cancelar campanha em andamento

`POST /v1/campaigns/{id}/cancel`

```bash
curl -X POST https://api.ararahq.com/v1/campaigns/7b3789fb-.../cancel \
  -H "Authorization: Bearer ara_live_xxx"
```

Marca como `CANCELLED`. Envios já disparados continuam processando no provider — só evita as próximas mensagens da fila.

## Erros

| HTTP | Quando |
|---|---|
| `400` | Body inválido (`name`/`templateName`/`contacts` ausente, `contacts` vazio, ou `Idempotency-Key` faltando) |
| `401` | API key inválida |
| `402` | Saldo da wallet abaixo do custo estimado |
| `403` | `PLAN_FEATURE_LOCKED` (plano não inclui campanhas) ou `NO_DEDICATED_NUMBER` |
| `422` | Template não existe, não aprovado, ou variáveis incompatíveis (vem em formato simplificado `{ error, message, diagnostic }`) |

Ver [Erros](/erros) pro formato completo.
