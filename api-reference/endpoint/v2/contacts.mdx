Seu CRM leve. Cada contato e identificado pelo `phone`, com nome, email, tags e atributos customizaveis.

**Base URL:** `https://new-api.ararahq.com/api/v2`

---

## Criar ou atualizar um contato

`POST /v2/contacts`

E um **upsert**: se o telefone ja existe, os dados sao sobrescritos. Se nao, e criado.

<CodeGroup>

```bash cURL
curl -X POST https://new-api.ararahq.com/api/v2/contacts \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "5511999999999",
    "name": "Joao Silva",
    "email": "joao@exemplo.com",
    "tags": ["cliente-vip"],
    "attributes": {
      "plan": "pro",
      "source": "landing-page"
    }
  }'
```

```javascript Node.js
await fetch("https://new-api.ararahq.com/api/v2/contacts", {
  method: "POST",
  headers: {
    "Authorization": "Bearer ara_live_xxx",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    phone: "5511999999999",
    name: "Joao Silva",
    email: "joao@exemplo.com",
    tags: ["cliente-vip"],
    attributes: { plan: "pro", source: "landing-page" },
  }),
});
```

```python Python
import requests

requests.post(
    "https://new-api.ararahq.com/api/v2/contacts",
    headers={"Authorization": "Bearer ara_live_xxx"},
    json={
        "phone": "5511999999999",
        "name": "Joao Silva",
        "email": "joao@exemplo.com",
        "tags": ["cliente-vip"],
        "attributes": {"plan": "pro", "source": "landing-page"},
    },
)
```

</CodeGroup>

Resposta (`201 Created`):

```json
{
  "id": "1f2e3d4c-5b6a-7890-abcd-ef1234567890",
  "phone": "5511999999999",
  "name": "Joao Silva",
  "email": "joao@exemplo.com",
  "tags": ["cliente-vip"],
  "attributes": { "plan": "pro", "source": "landing-page" },
  "createdAt": "2026-04-13T18:30:00Z",
  "updatedAt": "2026-04-13T18:30:00Z"
}
```

### Parametros

| Campo | Tipo | Obrigatorio | Descricao |
|-------|------|-------------|-----------|
| `phone` | string | sim | Telefone E.164 sem `+` (`5511999999999`). Identificador unico. |
| `name` | string | nao | Nome do contato. |
| `email` | string | nao | Email do contato. |
| `tags` | string[] | nao | Lista de tags. Max 20. |
| `attributes` | object | nao | Dados customizados chave-valor. |

## Buscar contato por telefone

`GET /v2/contacts/{phone}`

```bash
curl https://new-api.ararahq.com/api/v2/contacts/5511999999999 \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta (`200 OK`) — mesmo formato do POST. Se nao existir, retorna `404`.

## Listar contatos

`GET /v2/contacts`

```bash
curl "https://new-api.ararahq.com/api/v2/contacts?query=joao&page=0&size=20" \
  -H "Authorization: Bearer ara_live_xxx"
```

### Query params

| Param | Tipo | Default | Descricao |
|-------|------|---------|-----------|
| `query` | string | — | Busca por nome, telefone ou email. |
| `page` | int | `0` | Pagina (base 0). |
| `size` | int | `20` | Itens por pagina. |

Resposta (`200 OK`):

```json
{
  "data": [
    {
      "id": "1f2e3d4c-5b6a-7890-abcd-ef1234567890",
      "phone": "5511999999999",
      "name": "Joao Silva",
      "email": "joao@exemplo.com",
      "tags": ["cliente-vip"],
      "attributes": { "plan": "pro" },
      "createdAt": "2026-04-13T18:30:00Z",
      "updatedAt": "2026-04-13T18:30:00Z"
    }
  ],
  "pagination": {
    "page": 0,
    "size": 20,
    "totalElements": 1,
    "totalPages": 1
  }
}
```

## Importar em lote

`POST /v2/contacts/batch`

Crie ou atualize ate **10.000 contatos** em uma unica chamada. Cada item segue o mesmo formato do POST individual.

```bash
curl -X POST https://new-api.ararahq.com/api/v2/contacts/batch \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "contacts": [
      { "phone": "5511999999999", "name": "Joao Silva", "tags": ["abril-2026"] },
      { "phone": "5511888888888", "name": "Maria Santos", "tags": ["abril-2026"] },
      { "phone": "5511777777777", "name": "Carlos Oliveira", "tags": ["abril-2026"] }
    ]
  }'
```

### Parametros

| Campo | Tipo | Descricao |
|-------|------|-----------|
| `contacts` | array | Entre 1 e 10.000 contatos. |

Resposta (`200 OK`):

```json
{
  "imported": 3,
  "failed": 0
}
```

| Campo | Descricao |
|-------|-----------|
| `imported` | Contatos criados ou atualizados com sucesso. |
| `failed` | Contatos que falharam na validacao (ex: telefone vazio). |

<Note>
  **Partial success:** se parte dos contatos falhar, os demais ainda sao processados. O retorno informa quantos passaram e quantos falharam.
</Note>

---

## Erros

| HTTP | Quando |
|------|--------|
| `400` | `phone` ausente ou `tags` com mais de 20 itens. |
| `401` | Chave invalida. |
| `404` | Contato nao encontrado no GET. |
| `422` | Lote com menos de 1 ou mais de 10.000 contatos. |
