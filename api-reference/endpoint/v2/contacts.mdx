Os contatos são o CRM leve da Arara — phone como chave única, com nome, email e atributos customizáveis. Útil pra segmentar campanhas e enriquecer conversas com contexto.

**Base URL:** `https://api.ararahq.com`

<Note>
  A criação individual de contato (`POST /v1/contacts`) **não existe**. Pra inserir um contato isolado, use o endpoint batch com array de 1 item.
</Note>

## Listar contatos

`GET /v1/contacts`

```bash
curl "https://api.ararahq.com/v1/contacts?q=joão&page=0&size=20" \
  -H "Authorization: Bearer ara_live_xxx"
```

Query params:

| Param | Tipo | Default | Descrição |
|---|---|---|---|
| `q` | string | — | Busca por nome, telefone ou email |
| `page` | integer | `0` | Página |
| `size` | integer | `20` | Itens por página |

Resposta:

```json
{
  "contacts": [
    {
      "id": "1f2e3d4c-5b6a-7890-abcd-ef1234567890",
      "name": "João Silva",
      "phone": "+5511999999999",
      "email": "joao@exemplo.com",
      "attributes": {
        "plan": "pro",
        "ltv": 2400
      },
      "createdAt": "2026-05-18T14:30:00Z"
    }
  ],
  "total": 1,
  "page": 0,
  "size": 20,
  "totalPages": 1
}
```

## Buscar contato por telefone

`GET /v1/contacts/{phone}`

Phone em E.164 **com `+`** no path (sem prefixo `whatsapp:`):

```bash
curl https://api.ararahq.com/v1/contacts/+5511999999999 \
  -H "Authorization: Bearer ara_live_xxx"
```

Não existir retorna `404`.

## Importar em lote (upsert)

`POST /v1/contacts/batch`

Cria ou atualiza contatos. Body é **array direto** (não wrapped em objeto):

```bash
curl -X POST https://api.ararahq.com/v1/contacts/batch \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "name": "João Silva",
      "phone": "+5511999999999",
      "email": "joao@exemplo.com",
      "attributes": { "plan": "pro", "ltv": 2400 }
    },
    {
      "name": "Maria Santos",
      "phone": "+5511888888888"
    }
  ]'
```

Resposta (`200 OK`):

```json
{
  "importId": "fa1b2c3d-4567-89ab-cdef-0123456789ab",
  "created": 1,
  "updated": 1,
  "skipped": 0,
  "errors": []
}
```

Comportamento upsert:

- Phone **já existe** → atualiza campos enviados (`updated++`)
- Phone **novo** → cria novo contato (`created++`)
- Phone **inválido** ou **vazio** → entra em `errors[]` com motivo, `skipped++`

```json
{
  "importId": "fa1b2c3d-...",
  "created": 8,
  "updated": 2,
  "skipped": 1,
  "errors": [
    { "index": 7, "phone": "+5511invalid", "reason": "INVALID_PHONE_NUMBER" }
  ]
}
```

## Estatísticas

`GET /v1/contacts/stats`

```bash
curl https://api.ararahq.com/v1/contacts/stats \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta é mapa `String → Long` com contadores agregados:

```json
{
  "totalContacts": 12450,
  "withActiveConversation": 312,
  "optedOut": 87
}
```

Os campos retornados podem variar conforme features ativas na sua org.

## Campos do contato

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | UUID | Identificador interno do contato |
| `phone` | string | E.164 (`+5511999999999`). **Chave única** na org |
| `name` | string | Nome — **obrigatório** ao criar/atualizar |
| `email` | string \| null | Email — opcional, útil pra atribuição cross-channel |
| `attributes` | object \| null | Chave-valor livre. Acessível em templates via Brain |
| `createdAt` | ISO 8601 | Quando foi criado |

`ContactRequest` (no batch) valida:
- `name`: obrigatório, max 255 chars
- `phone`: obrigatório, regex E.164 (`^\+[1-9]\d{6,14}$`)
- `email`: opcional, max 255 chars
- `attributes`: mapa livre

## Opt-out — comportamento

Contato em opt-out **continua na base** mas:

- `POST /v1/messages` com template `MARKETING` retorna `422 NUMBER_BLOCKED`
- Template `UTILITY` ainda passa (notificação transacional é permitida)

Pra registrar opt-out programaticamente, use o endpoint de opt-outs (não documentado aqui ainda). Ver [Compliance](/concepts/compliance).

## Erros

Validação de body (`@Valid`) e autenticação usam o envelope **simplificado** do Spring (sem `error.code`) — ramifique pelo status HTTP. Ver [Erros](/erros).

| HTTP | Envelope | Quando |
|---|---|---|
| `400` | simplificado (sem `code`) | Telefone fora do E.164 ou `name`/`phone` ausente (`@Valid`). |
| `400` | `INVALID_JSON` | JSON malformado. |
| `401` | simplificado (sem `code`) | Chave inválida, revogada ou expirada. |
| `404` | — | Phone não existe na sua org (no GET). |
| `422` | `NUMBER_BLOCKED` | Destinatário em opt-out (no envio pra ele). |
