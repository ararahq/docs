Uma unica API para WhatsApp e email. Envio imediato ou agendado, template ou texto livre.

**Base URL:** `https://new-api.ararahq.com/api/v2`

---

## Envio rapido

Cole no terminal, troque `ara_live_xxx` pela sua chave e o numero pelo seu. Pronto.

```bash
curl -X POST https://new-api.ararahq.com/api/v2/messages \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "5511999999999",
    "channel": "whatsapp",
    "template": "hello_world"
  }'
```

Resposta (`202 Accepted`):

```json
{
  "id": "ara_msg_k9Bf2XqP",
  "channel": "whatsapp",
  "status": "PENDING",
  "to": "5511999999999",
  "from": "5511988887777",
  "template": "hello_world",
  "cost": null,
  "createdAt": "2026-04-13T18:30:00Z",
  "deliveredAt": null
}
```

Guarde o `id` — e por ele que voce consulta status e custo depois.

## Enviar template com variaveis

As variaveis sao um **array** na ordem dos placeholders (`{{1}}`, `{{2}}`, `{{3}}`...).

<CodeGroup>

```bash cURL
curl -X POST https://new-api.ararahq.com/api/v2/messages \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "5511999999999",
    "channel": "whatsapp",
    "template": "pedido_confirmado",
    "variables": ["Joao", "12345"]
  }'
```

```javascript Node.js
await fetch("https://new-api.ararahq.com/api/v2/messages", {
  method: "POST",
  headers: {
    "Authorization": "Bearer ara_live_xxx",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    to: "5511999999999",
    channel: "whatsapp",
    template: "pedido_confirmado",
    variables: ["Joao", "12345"],
  }),
});
```

```python Python
import requests

requests.post(
    "https://new-api.ararahq.com/api/v2/messages",
    headers={"Authorization": "Bearer ara_live_xxx"},
    json={
        "to": "5511999999999",
        "channel": "whatsapp",
        "template": "pedido_confirmado",
        "variables": ["Joao", "12345"],
    },
)
```

</CodeGroup>

Template `Ola {{1}}, seu pedido #{{2}} foi confirmado.` vira `Ola Joao, seu pedido #12345 foi confirmado.`

## Responder com texto livre

Dentro da janela de 24h de uma conversa, mande `body` em vez de `template`:

```bash
curl -X POST https://new-api.ararahq.com/api/v2/messages \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "5511999999999",
    "channel": "whatsapp",
    "body": "Ola! Como posso ajudar?"
  }'
```

## Enviar email

Mesmo endpoint, trocando `channel`:

```bash
curl -X POST https://new-api.ararahq.com/api/v2/messages \
  -H "Authorization: Bearer ara_live_xxx" \

  -d '{
    "to": "cliente@exemplo.com",
    "channel": "email",
    "template": "welcome_email",
    "variables": ["Joao"]
  }'
```

## Agendar envio

Adicione `scheduled_at` com data/hora ISO 8601 em UTC:

```json
{
  "to": "5511999999999",
  "channel": "whatsapp",
  "template": "lembrete",
  "variables": ["14h"],
  "scheduled_at": "2026-04-14T17:00:00Z"
}
```

A mensagem fica em `SCHEDULED` ate a hora chegar.

## Consultar status

`GET /v2/messages/{id}`

```bash
curl https://new-api.ararahq.com/api/v2/messages/ara_msg_k9Bf2XqP \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta (`200 OK`):

```json
{
  "id": "ara_msg_k9Bf2XqP",
  "channel": "whatsapp",
  "status": "DELIVERED",
  "to": "5511999999999",
  "from": "5511988887777",
  "template": "pedido_confirmado",
  "cost": 0.1600,
  "createdAt": "2026-04-13T18:30:00Z",
  "deliveredAt": "2026-04-13T18:30:05Z"
}
```

## Parametros do POST

| Campo | Tipo | Obrigatorio | Descricao |
|-------|------|-------------|-----------|
| `to` | string | sim | WhatsApp: numero E.164 sem `+` (`5511999999999`). Email: endereco completo. |
| `channel` | string | sim | `whatsapp` ou `email`. |
| `template` | string | * | Nome do template pre-aprovado. |
| `body` | string | * | Texto livre (responder dentro da janela de 24h). |
| `variables` | string[] | nao | Array ordenado dos valores dos placeholders `{{1}}`, `{{2}}`... Max 50. |
| `scheduled_at` | string (ISO 8601) | nao | Envio agendado em UTC. |

<Note>
  **`template` OU `body`, nunca os dois.** Template inicia conversa, body responde dentro da janela de 24h.
</Note>

---

## Campos da resposta

| Campo | Tipo | Descricao |
|-------|------|-----------|
| `id` | string | ID unico (`ara_msg_...`). Use em webhooks e consultas. |
| `channel` | string | `whatsapp` ou `email`. |
| `status` | string | Status atual (ver tabela abaixo). |
| `to` | string | Destinatario. |
| `from` | string | Remetente configurado na sua organizacao. |
| `template` | string \| null | Nome do template. `null` se for texto livre. |
| `subject` | string \| null | Assunto do email com variaveis aplicadas. So aparece em `channel=email`. |
| `cost` | number \| null | Custo em BRL. `null` ate a mensagem ser cobrada. |
| `createdAt` | string (ISO 8601) | Quando foi criada. |
| `deliveredAt` | string (ISO 8601) \| null | Quando chegou no destinatario. |

## Status

```
PENDING -> PROCESSING -> SENT_TO_PROVIDER -> SENT -> DELIVERED -> READ
                                                             `-> FAILED
SCHEDULED --^                                                 CANCELED
```

| Status | Significado |
|--------|-------------|
| `PENDING` | Criada, aguardando processamento. |
| `SCHEDULED` | Agendada, aguardando o horario. |
| `PROCESSING` | Worker pegou a mensagem. |
| `SENT_TO_PROVIDER` | Entregue ao provider (Meta/email). |
| `SENT` | Provider confirmou o envio. |
| `DELIVERED` | Chegou no destinatario. |
| `READ` | Foi lida (so WhatsApp). |
| `FAILED` | Falhou definitivamente. |
| `CANCELED` | Cancelada antes do envio. |

Para saber em tempo real, use [webhooks](/api-reference/events) em vez de polling.

## Idempotencia

Para garantir que um retry nao duplique mensagem, envie o header `Idempotency-Key`:

```bash
-H "Idempotency-Key: 7f8a1c2d-4e5f-6789-abcd-ef0123456789"
```

Requisicoes com a mesma chave retornam a resposta original sem criar nova mensagem.

## Erros

| HTTP | Quando |
|------|--------|
| `400` | Payload invalido, canal nao suportado, ou sem `template`/`body`. |
| `401` | Chave invalida ou ausente. |
| `402` | Saldo insuficiente. |
| `404` | Mensagem nao encontrada (no GET). |
| `422` | Template inexistente ou numero de variaveis errado. |
| `429` | Rate limit. Aplique backoff exponencial. |

Formato padrao:

```json
{
  "error": {
    "code": "INVALID_CHANNEL",
    "message": "Canal 'sms' nao e suportado. Canais validos: whatsapp, email",
    "details": {}
  }
}
```

## Migrando da V1

<Warning>
  **V1 esta deprecada** (sunset **09/2026**). Use V2 para integracoes novas.
</Warning>

| | V1 | V2 |
|---|---|---|
| Destinatario | `receiver: "whatsapp:+5511999999999"` | `to: "5511999999999"` + `channel` |
| Template | `templateName` | `template` |
| Variaveis | `templateVariables: [...]` | `variables: [...]` |
| Canais | So WhatsApp | WhatsApp **e** Email |
| Agendamento | `scheduledAt` | `scheduled_at` |
