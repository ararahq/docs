---
title: 'Templates'
description: 'Crie e gerencie templates de mensagem para WhatsApp e Email.'
---

Templates sao as mensagens pre-aprovadas que voce pode disparar. WhatsApp exige aprovacao da Meta; email nao.

**Base URL:** `https://new-api.ararahq.com/api/v2`

---

## Criar um template WhatsApp

`POST /v2/templates`

```bash
curl -X POST https://new-api.ararahq.com/api/v2/templates \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "pedido_confirmado",
    "channel": "whatsapp",
    "category": "UTILITY",
    "language": "pt_BR",
    "body": "Ola {{1}}, seu pedido #{{2}} foi confirmado.",
    "variables": ["Joao", "12345"]
  }'
```

Resposta (`201 Created`):

```json
{
  "id": "a7f3c1e2-4b5d-6789-abcd-ef0123456789",
  "name": "pedido_confirmado",
  "channel": "whatsapp",
  "category": "UTILITY",
  "status": "PENDING",
  "body": "Ola {{1}}, seu pedido #{{2}} foi confirmado.",
  "language": "pt_BR",
  "createdAt": "2026-04-13T18:30:00Z"
}
```

### Parametros

| Campo | Tipo | Obrigatorio | Descricao |
|-------|------|-------------|-----------|
| `name` | string | sim | Nome unico em snake_case (sem espacos/acentos). |
| `channel` | string | sim | `whatsapp` ou `email`. |
| `category` | string | sim | `UTILITY` ou `MARKETING`. Default `UTILITY`. |
| `body` | string | sim | Texto do template. Use `{{1}}`, `{{2}}`... para variaveis. |
| `language` | string | nao | Codigo do idioma. Default `pt_BR`. |
| `footer` | string | nao | Rodape opcional. |
| `variables` | string[] | nao | Valores de exemplo para cada placeholder (usado na aprovacao da Meta). |
| `buttons` | array | nao | Lista de botoes (ver abaixo). |

<Note>
  Apos criar, o template entra em `status: PENDING` ate a Meta aprovar (geralmente 1-5 minutos). So mensagens com template `APPROVED` podem ser disparadas.
</Note>

### Com botoes

```json
{
  "name": "pesquisa_satisfacao",
  "channel": "whatsapp",
  "category": "MARKETING",
  "body": "O que achou do atendimento?",
  "buttons": [
    { "type": "QUICK_REPLY", "text": "Otimo" },
    { "type": "QUICK_REPLY", "text": "Bom" },
    { "type": "QUICK_REPLY", "text": "Pode melhorar" }
  ]
}
```

| Campo do botao | Tipo | Descricao |
|----------------|------|-----------|
| `type` | string | `QUICK_REPLY`, `URL` ou `PHONE_NUMBER`. |
| `text` | string | Texto exibido no botao. |
| `url` | string | Obrigatorio quando `type=URL`. |

## Criar um template de Email

`POST /v2/templates` com `channel: "email"`.

```bash
curl -X POST https://new-api.ararahq.com/api/v2/templates \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "welcome_email",
    "channel": "email",
    "subject": "Bem-vindo, {{1}}!",
    "htmlContent": "<h1>Ola {{1}}</h1><p>Sua conta foi criada.</p>"
  }'
```

Resposta (`201 Created`):

```json
{
  "id": "b8c9d0e1-2f3a-4b5c-6d7e-8f9012345678",
  "name": "welcome_email",
  "channel": "email",
  "category": "TRANSACTIONAL",
  "status": "APPROVED",
  "subject": "Bem-vindo, {{1}}!",
  "createdAt": "2026-04-13T18:30:00Z"
}
```

| Campo | Obrigatorio | Descricao |
|-------|-------------|-----------|
| `name` | sim | Nome unico. |
| `channel` | sim | Deve ser `email`. |
| `subject` | sim | Assunto do email. Pode conter `{{1}}`, `{{2}}`... |
| `htmlContent` | sim | HTML do email. |
| `designJson` | nao | JSON do editor visual (se estiver usando). |

<Note>
  Templates de email nao passam por aprovacao externa â€” ja sao criados com `status: APPROVED` e podem ser usados imediatamente.
</Note>

## Listar templates

`GET /v2/templates`

Retorna um **array simples** (sem paginacao) com todos os templates da organizacao.

```bash
curl "https://new-api.ararahq.com/api/v2/templates?channel=whatsapp" \
  -H "Authorization: Bearer ara_live_xxx"
```

### Query params

| Param | Tipo | Descricao |
|-------|------|-----------|
| `channel` | string | Opcional. `whatsapp` ou `email`. Omitir retorna ambos. |

Resposta (`200 OK`):

```json
[
  {
    "id": "a7f3c1e2-4b5d-6789-abcd-ef0123456789",
    "name": "pedido_confirmado",
    "channel": "whatsapp",
    "category": "UTILITY",
    "status": "APPROVED",
    "body": "Ola {{1}}, seu pedido #{{2}} foi confirmado.",
    "language": "pt_BR"
  },
  {
    "id": "b8c9d0e1-2f3a-4b5c-6d7e-8f9012345678",
    "name": "welcome_email",
    "channel": "email",
    "category": "TRANSACTIONAL",
    "status": "APPROVED",
    "subject": "Bem-vindo, {{1}}!"
  }
]
```

## Analytics do template

`GET /v2/templates/{templateId}/analytics`

Metricas de entrega, leitura e cliques em links encurtados (SmartLinks) do template nos ultimos 30 dias.

```bash
curl https://new-api.ararahq.com/api/v2/templates/a7f3c1e2-4b5d-6789-abcd-ef0123456789/analytics \
  -H "Authorization: Bearer ara_live_xxx"
```

## Status do template

| Status | Significado |
|--------|-------------|
| `PENDING` | Enviado a Meta, aguardando aprovacao. |
| `APPROVED` | Aprovado. Pronto para disparar. |
| `REJECTED` | Rejeitado. Veja o motivo nos logs da Meta. |

Templates de email comecam direto em `APPROVED`.

---

## Erros

| HTTP | Quando |
|------|--------|
| `400` | `channel` fora de `whatsapp`/`email`, campos obrigatorios ausentes. |
| `401` | Chave invalida. |
| `404` | Template nao encontrado (no GET analytics). |
| `409` | `name` ja existe na organizacao. |
| `422` | Body invalido para a Meta (ex: variaveis em formato errado). |
