Templates HSM (Highly Structured Messages) são as mensagens pré-aprovadas que você pode mandar fora da janela 24h. Esta página cobre os endpoints REST. Pro **conceito** (categorias, regras de aprovação, lint, política), ver [Templates HSM](/concepts/templates) e [Política de templates](/templates/policy).

**Base URL:** `https://api.ararahq.com/api`

## Criar template

`POST /v1/templates`

```bash
curl -X POST https://api.ararahq.com/api/v1/templates \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "pedido_confirmado",
    "category": "UTILITY",
    "language": "pt_BR",
    "body": "Oi {{1}}, seu pedido #{{2}} foi confirmado!\nTotal: R$ {{3}}",
    "footer": "Acompanhe pelo nosso site",
    "samples": {
      "1": "João",
      "2": "ARA-12345",
      "3": "147,90"
    }
  }'
```

Resposta (`201 Created`):

```json
{
  "id": "a7f3c1e2-4b5d-6789-abcd-ef0123456789",
  "name": "pedido_confirmado",
  "formattedName": "pedido_confirmado",
  "category": "UTILITY",
  "originalCategory": "UTILITY",
  "language": "pt_BR",
  "providerName": "TWILIO",
  "providerTemplateId": "HXabc123def456...",
  "providerStatus": "PENDING",
  "bodyPreview": "Oi {{1}}, seu pedido #{{2}} foi confirmado!\nTotal: R$ {{3}}",
  "structureJson": { },
  "usageGuide": {
    "message": "Use o objeto 'variables' no envio da mensagem para substituir os placeholders.",
    "endpoint": "POST /v1/messages"
  },
  "variablesSchema": {
    "1": "João",
    "2": "ARA-12345",
    "3": "147,90"
  },
  "createdAt": "2026-05-18T14:30:00Z",
  "updatedAt": null
}
```

`providerStatus: PENDING` até a Meta aprovar (1-72h dependendo da categoria — UTILITY mais rápido).

<Note>
  IDs de template são **UUID puro** (`a7f3c1e2-4b5d-...`), não prefixados.
</Note>

### Parâmetros

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `name` | string | sim | Único na org. snake_case, sem espaços/acentos. |
| `category` | string | sim | `UTILITY`, `MARKETING` ou `AUTHENTICATION`. |
| `body` | string | sim | Texto com placeholders `{{1}}`, `{{2}}`... Max 1024 chars. |
| `language` | string | não | `pt_BR` (default), `en_US`, `es_ES`. |
| `header` | string | não | Cabeçalho — texto ou URL de mídia. |
| `headerType` | string | não | `text`, `media`, `document`. |
| `footer` | string | não | Rodapé. Max 60 chars. Sem variáveis. |
| `samples` | `Map<string, string>` | não | Mapa de exemplo por número da variável. Meta usa pra revisar. Ex: `{"1": "João"}`. |
| `variableExamples` | string[] | não | Legado, compat. Array de exemplos posicionais — use `samples` em produção. |
| `buttons` | array | não | Até 3 botões (ver abaixo). |
| `carouselCards` | array | não | 2-10 cards pra template carrossel (ver abaixo). |
| `otpButtonType` | string | não | `COPY_CODE` ou `ONE_TAP` (só pra `AUTHENTICATION`). |
| `otpPackageName` | string | não | Android package name. Obrigatório se `otpButtonType: ONE_TAP`. |
| `otpAppSignature` | string | não | Android app signature hash. Obrigatório se `otpButtonType: ONE_TAP`. |

<Warning>
  `samples` é um **mapa indexado pelo número da variável** (`{"1": "...", "2": "..."}`), não array. Cliente que enviar array recebe `400`.
</Warning>

### Com botões

```json
{
  "name": "pesquisa_satisfacao",
  "category": "MARKETING",
  "language": "pt_BR",
  "body": "Como foi sua experiência?",
  "samples": {},
  "buttons": [
    { "type": "QUICK_REPLY", "text": "Ótimo" },
    { "type": "QUICK_REPLY", "text": "Bom" },
    { "type": "QUICK_REPLY", "text": "Pode melhorar" }
  ]
}
```

Tipos de botão:

| `type` | Comportamento |
|---|---|
| `QUICK_REPLY` | Resposta rápida. Volta como webhook `message.received` com `type: button`. |
| `URL` | Abre URL. Pode incluir variável: `https://meusite.com/{{1}}`. |
| `PHONE_NUMBER` | Liga pra número fixo (não pode ser variável). |
| `SMART_LINK` | Variante Arara: URL curta com tracking. Ver [Smart Links](/smart-links). |
| `COPY_CODE` | Copia código pra clipboard. Obrigatório em AUTHENTICATION. |

### Carrossel (2-10 cards)

```json
{
  "name": "produtos_destaque",
  "category": "MARKETING",
  "language": "pt_BR",
  "body": "Confira nossas ofertas:",
  "samples": {},
  "carouselCards": [
    {
      "mediaUrl": "https://meusite.com/produto-1.jpg",
      "body": "Camiseta básica - R$ 79",
      "buttons": [
        { "type": "URL", "text": "Comprar", "url": "https://meusite.com/p/1" }
      ]
    },
    {
      "mediaUrl": "https://meusite.com/produto-2.jpg",
      "body": "Tênis corrida - R$ 299",
      "buttons": [
        { "type": "URL", "text": "Comprar", "url": "https://meusite.com/p/2" }
      ]
    }
  ]
}
```

<Note>
  Carousel exige: 2-10 cards, **todos com o mesmo número e tipo de botões**, cada card com mídia (imagem ou vídeo).
</Note>

### OTP (AUTHENTICATION)

Templates `AUTHENTICATION` exigem botão `COPY_CODE` ou `ONE_TAP`:

```json
{
  "name": "otp_login",
  "category": "AUTHENTICATION",
  "language": "pt_BR",
  "body": "Seu código de verificação é {{1}}",
  "samples": { "1": "123456" },
  "otpButtonType": "ONE_TAP",
  "otpPackageName": "com.minhaempresa.app",
  "otpAppSignature": "abc123def456..."
}
```

## Listar templates

`GET /v1/templates`

Retorna **array direto** (sem paginação). Aceita filtros opcionais por nome ou status.

```bash
curl "https://api.ararahq.com/api/v1/templates?status=APPROVED" \
  -H "Authorization: Bearer ara_live_xxx"
```

Query params:

| Param | Tipo | Descrição |
|---|---|---|
| `name` | string | Filtra por nome exato (case-sensitive). |
| `status` | string | Filtra por providerStatus (`APPROVED`, `PENDING`, `REJECTED`, `PAUSED`). |

Resposta (`200 OK`):

```json
[
  {
    "id": "a7f3c1e2-4b5d-6789-abcd-ef0123456789",
    "name": "pedido_confirmado",
    "formattedName": "pedido_confirmado",
    "category": "UTILITY",
    "language": "pt_BR",
    "providerName": "TWILIO",
    "providerTemplateId": "HXabc...",
    "providerStatus": "APPROVED",
    "bodyPreview": "Oi {{1}}, seu pedido #{{2}} foi confirmado!",
    "createdAt": "2026-05-18T14:30:00Z"
  }
]
```

## Obter template específico

`GET /v1/templates/{id}`

```bash
curl https://api.ararahq.com/api/v1/templates/a7f3c1e2-4b5d-6789-abcd-ef0123456789 \
  -H "Authorization: Bearer ara_live_xxx"
```

Path param `id` é **UUID**.

## Status de aprovação

`GET /v1/templates/{id}/status`

Consulta a Twilio (source of truth) e atualiza o status local. Útil quando você acha que algo mudou e quer status fresh.

```bash
curl https://api.ararahq.com/api/v1/templates/{id}/status \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta:

```json
{
  "status": "APPROVED",
  "rejectionReason": null,
  "category": "UTILITY"
}
```

Valores possíveis de `status`:

| Status | Significado |
|---|---|
| `PENDING` | Em revisão Meta |
| `APPROVED` | Aprovado — pronto pra disparar |
| `REJECTED` | Rejeitado. `rejectionReason` traz motivo da Meta |
| `PAUSED` | Pausado por alta taxa de bloqueio. Modifique e ressubmeta |

Pra acompanhar **em tempo real sem polling**, escute o webhook `template.updated` (o campo `status` em `data` traz `APPROVED`, `REJECTED`, `PAUSED`, etc). Ver [Eventos](/webhooks/events).

## Analytics

`GET /v1/templates/{id}/analytics`

```bash
curl "https://api.ararahq.com/api/v1/templates/{id}/analytics?period=30d" \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta:

```json
{
  "templateId": "a7f3c1e2-4b5d-6789-abcd-ef0123456789",
  "templateName": "pedido_confirmado",
  "period": "30d",
  "sent": 1250,
  "delivered": 1230,
  "read": 890,
  "failed": 10,
  "deliveryRate": "98.4",
  "readRate": "71.2"
}
```

Query params: `period` = `7d`, `30d`, `90d` (default `30d`).

<Note>
  `deliveryRate` e `readRate` voltam como **string** formatada com 1 casa decimal (ex: `"98.4"`). Parse no cliente como número se quiser fazer cálculo.
</Note>

## Deletar template

`DELETE /v1/templates/{id}`

```bash
curl -X DELETE https://api.ararahq.com/api/v1/templates/a7f3c1e2-4b5d-6789-abcd-ef0123456789 \
  -H "Authorization: Bearer ara_live_xxx"
```

Resposta (`204 No Content`) — sem body.

<Warning>
  Deletar template é **irreversível**. Histórico de envios continua nos logs, mas o template some da listagem e não pode ser usado em novos disparos.
</Warning>

## Limites

| Plano | Templates ativos |
|---|---|
| FREE | 3 |
| ESSENTIAL+ | sem limite prático |

Tentar criar acima do limite no plano FREE retorna `403 PLAN_LIMIT_REACHED` com indicação pra fazer upgrade pra Essential.

## Conta em modo TESTING

Templates **só podem ser criados em conta com `accountStatus: ACTIVE`** (modo produção). Conta ainda em onboarding (`TESTING`) recebe:

```json
{
  "error": "Recurso disponível apenas no Modo Produção.",
  "message": "Por favor, ative sua conta para criar templates."
}
```

Status HTTP `403`.

## Erros

| HTTP | Quando |
|---|---|
| `400` | `name`/`category`/`language`/`body` ausente; categoria fora de `UTILITY/MARKETING/AUTHENTICATION` |
| `401` | Chave inválida ou ausente |
| `403` | Conta ainda em TESTING; ou plano limit atingido |
| `404` | Template não existe |
| `422` | Body viola política Meta (ver [Política](/templates/policy)) |
