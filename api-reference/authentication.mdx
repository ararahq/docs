## API Keys

Todas as requisicoes exigem uma API key no header `Authorization`:

```bash
Authorization: Bearer ara_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Ambientes

A Arara possui dois ambientes, diferenciados pelo prefixo da chave:

| Prefixo | Ambiente | Comportamento |
|---------|----------|---------------|
| `ara_live_` | Producao | Mensagens entregues. Creditos consumidos. |
| `ara_test_` | Sandbox | Mensagens simuladas. Sem custo. |

Use chaves `ara_test_` para desenvolvimento e testes. Troque para `ara_live_` quando estiver pronto para producao.

## Como obter sua chave

### Via Dashboard

1. Acesse o [Dashboard](https://www.ararahq.com/dashboard)
2. No primeiro login, uma chave `ara_live_` e gerada automaticamente

<Warning>
  A chave e exibida apenas uma vez. Guarde-a imediatamente em um local seguro (ex: variavel de ambiente, secrets manager). Voce nao podera ve-la novamente.
</Warning>

### Via API

Voce pode criar chaves adicionais programaticamente:

```bash
curl -X POST https://new-api.ararahq.com/api/v2/api-keys \
  -H "Authorization: Bearer ara_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "backend-production",
    "mode": "LIVE"
  }'
```

Resposta `201 Created`:

```json
{
  "id": "ara_key_abc123",
  "name": "backend-production",
  "key": "ara_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "mode": "LIVE",
  "createdAt": "2026-04-04T14:30:00Z"
}
```

<Info>
  O campo `key` so aparece na resposta de criacao. Nas listagens, a chave e exibida mascarada (ex: `ara_live_****xxxx`).
</Info>

Consulte a referencia completa em [API Keys](/api-reference/endpoint/v2/api-keys).

## Seguranca

- Nunca exponha sua chave em codigo frontend, repositorios publicos ou logs
- Use variaveis de ambiente (`ARARA_API_KEY`) em vez de valores hardcoded
- Crie chaves separadas para cada ambiente (dev, staging, producao)
- Revogue chaves comprometidas imediatamente via Dashboard ou API

## Erro de autenticacao

Se a chave for invalida, ausente ou revogada, a API retorna `401 Unauthorized`:

```json
{
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid or has been revoked."
  }
}
```

Se a chave for valida mas nao tiver permissao para o recurso, a API retorna `403 Forbidden`:

```json
{
  "error": {
    "code": "FORBIDDEN",
    "message": "Your API key does not have permission to access this resource."
  }
}
```
