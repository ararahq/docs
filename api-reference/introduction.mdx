## Base URL

Todas as requisicoes devem ser feitas para:

```
https://new-api.ararahq.com/api
```

Endpoints sao versionados no path (ex: `/v2/messages`, `/v2/templates`).

<Accordion title="Usando a V1?">
  A V1 continua disponivel em `https://api.ararahq.com/api/v1`. Se voce esta migrando, consulte a secao [V1 (Legacy)](/api-reference/endpoint/messages) para a documentacao dos endpoints antigos.
</Accordion>

## Autenticacao

Todas as requisicoes exigem um header `Authorization` com sua API key:

```bash
Authorization: Bearer ara_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Existem dois tipos de chave:

| Prefixo | Ambiente | Descricao |
|---------|----------|-----------|
| `ara_live_` | Producao | Mensagens entregues a destinatarios reais. Creditos consumidos. |
| `ara_test_` | Sandbox | Mensagens simuladas. Nenhum credito consumido. |

Consulte a pagina de [Autenticacao](/api-reference/authentication) para detalhes sobre criacao e revogacao de chaves.

## Formato de resposta

Todas as respostas sao em **JSON**. Datas seguem o padrao **ISO 8601 (UTC)**.

### Resposta de sucesso (objeto)

```json
{
  "id": "ara_msg_k9Bf2XqP",
  "status": "QUEUED",
  "createdAt": "2026-04-04T14:30:00Z"
}
```

### Resposta de sucesso (lista paginada)

Toda lista retorna o mesmo envelope:

```json
{
  "data": [
    { "id": "ara_tpl_abc123", "name": "hello_world_v2" }
  ],
  "pagination": {
    "page": 0,
    "size": 20,
    "totalElements": 150,
    "totalPages": 8
  }
}
```

### Resposta de erro

Todos os erros seguem o mesmo formato:

```json
{
  "error": {
    "code": "INSUFFICIENT_CREDITS",
    "message": "Your wallet does not have enough credits for this operation."
  }
}
```

Consulte a pagina de [Erros](/erros) para a lista completa de codigos.

## Paginacao

Endpoints que retornam listas aceitam os seguintes query parameters:

| Parametro | Tipo | Default | Descricao |
|-----------|------|---------|-----------|
| `page` | integer | `0` | Numero da pagina (zero-indexed). |
| `size` | integer | `20` | Itens por pagina (maximo 100). |

## Rate limits

Requisicoes que excedam o limite recebem `429 Too Many Requests`. O header `Retry-After` indica quantos segundos aguardar.

## Idempotencia

Endpoints `POST` que criam recursos aceitam o header `Idempotency-Key` para evitar duplicatas em retries:

```bash
Idempotency-Key: unique-request-id-123
```

Se uma requisicao com a mesma chave ja foi processada, a API retorna a resposta original sem criar um novo recurso.

## Codigos de status HTTP

| Codigo | Descricao | Quando |
|--------|-----------|--------|
| `200` | OK | Sucesso com body (ex: listar, consultar). |
| `201` | Created | Recurso criado com sucesso (ex: criar template). |
| `202` | Accepted | Requisicao aceita para processamento assincrono (ex: enviar mensagem). |
| `204` | No Content | Sucesso sem body (ex: deletar recurso). |
| `400` | Bad Request | Input invalido ou malformado. |
| `401` | Unauthorized | API key invalida ou ausente. |
| `403` | Forbidden | Autenticado, mas sem permissao para este recurso. |
| `404` | Not Found | Recurso nao encontrado. |
| `409` | Conflict | Conflito (duplicata, estado invalido). |
| `429` | Too Many Requests | Rate limit excedido. |
| `500` | Internal Server Error | Erro interno. Tente novamente ou contate suporte. |
